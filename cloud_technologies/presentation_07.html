1. VM проти Контейнерів

Яку проблему ми вирішуємо?

Головний біль розробника

"На моєму ноутбуці все працює ідеально! А на сервері чомусь помилка..."

Рішення: Упакувати саму програму і всі її залежності (бібліотеки) в єдиний монолітний ящик під назвою "Контейнер".

Відмінності

Віртуальна машина (VM) Контейнер (Docker)
  • Власна (гостьова) ОС.
  • Важить гігабайти.
  • Запускається хвилини.
  • Повна апаратна ізоляція.
  • Спільне ядро з хост-ОС. Немає власної ОС.
  • Важить мегабайти.
  • Запускається за секунди.
  • Ізоляція на рівні процесів.

2. Основи Docker

Стандарт де-факто у контейнерах

Образ (Image) vs Контейнер

  • Образ (Image) — це незмінний шаблон (креслення) вашої програми.
  • Контейнер — це "живий" запущений процес, створений із цього шаблону. З одного образу можна запустити 100 контейнерів.

Dockerfile

Це текстовий файл-інструкція, як зібрати образ:


# 1. Візьми за основу готовий Python
FROM python:3.11-slim

# 2. Скопіюй мій код
COPY . /app
WORKDIR /app

# 3. Встанови пакети
RUN pip install -r requirements.txt

# 4. При запуску - виконай ось це
CMD ["python", "server.py"]

3. Реєстр контейнерів

Де зберігати готові образи?

Container Registry

Аналог GitHub, але не для вихідного коду, а для готових зібраних Docker-образів.

  • Docker Hub: Найбільший публічний реєстр у світі.
  • Amazon ECR: Приватний і безпечний реєстр в хмарі AWS. Сам сканує образи на віруси та вразливості.

4. Оркестрація: Kubernetes

Навіщо нам оркестратор?

Проблема одного Docker

  • Docker чудово крутить 5 контейнерів на одному сервері.
  • Що робити, якщо у нас 5 000 контейнерів на 100 серверах?
  • Хто перезапустить контейнер, якщо він раптово "впаде"?
  • Хто автоматично додасть нові контейнери під час "Чорної п'ятниці"?

Для цього створили Kubernetes (K8s).

Основні поняття K8s

Термін Опис
Pod Мінімальна піщинка в K8s. Обгортка над одним або кількома контейнерами.
Deployment Правило: "Я хочу, щоб завжди працювало рівно 5 Pod'ів з моїм сайтом. Якщо один помре — сам створи новий".
Service Стабільна IP-адреса та балансувальник. Трафік з Інтернету йде на Service, а він вже роздає його на Pod'и.

5. Керовані хмарні сервіси

Не налаштовуйте K8s власноруч

Головні гравці

  • GKE (Google): Найкрутіший і найпросунутіший, адже саме Google придумала Kubernetes.
  • EKS (Amazon): Найпопулярніший для бізнесу, глибоко інтегрований з сервісами AWS.
  • AKS (Azure): Безкоштовний "мозок" (Control Plane), ви платите тільки за самі сервери-Worker'и.

* Для тих, кому K8s занадто складний, в Amazon існує "спрощена альтернатива" — сервіс ECS.

Висновки

  • Контейнери розв'язали проблему "на моєму комп'ютері працює", упакувавши залежності разом із кодом.
  • Docker швидший і легший за віртуальну машину (не містить власної ОС).
  • Kubernetes (K8s) життєво необхідний для керування сотнями контейнерів у production, забезпечуючи відмовостійкість.
  • Керовані хмарні сервіси (EKS, GKE) звільняють системних адміністраторів від пекельної роботи з налаштування K8s "з нуля".
