1. Що таке "хмарні обчислення"?

За визначенням NIST: це модель надання зручного мережевого доступу на вимогу до спільного пулу налаштовуваних обчислювальних ресурсів (мереж, серверів, сховищ, застосунків), що може бути швидко виділений з мінімальними зусиллями.

Простими словами: оренда ресурсів через Інтернет за моделлю "як послуга".

5 характеристик (NIST)

  • Самообслуговування на вимогу: оренда без людей-посередників.
  • Широкий мережевий доступ: з будь-якого пристрою.
  • Об'єднання в пул: ресурси спільні для багатьох (ізольовано).
  • Еластичність: миттєве масштабування.
  • Вимірюваність послуг: платиш за те, що використав.

Чому це вигідно?

  • 💵 Зниження витрат: нуль капітальних інвестицій (на сервери).
  • 📈 Швидке масштабування: адаптація до навантаження.
  • 🌍 Доступність: робота з будь-якої точки світу.
  • 🛡️ Надійність та Безпека: захист від провідних техно-гігантів.

2. Як ми до цього прийшли?

Від мейнфреймів до Cloud-first

Хронологія розвитку

  • 1970-ті: IBM VM/370 — перша комерційна віртуалізація.
  • 1999: Salesforce — поява першого SaaS.
  • 1990-2000-ті: Власні сервери компаній (On-premise).
  • 2006: Amazon запускає AWS (EC2, S3). Народження публічної хмари!
  • 2010: Запуск Microsoft Azure.
  • 2013-2014: Docker (контейнери) та AWS Lambda (Serverless).

3. Моделі хмарних послуг

IaaS, PaaS, SaaS

IaaS — Інфраструктура як послуга

🌥️ Що це: Провайдер дає "залізо", мережу та диски.

  • Ви самі встановлюєте ОС, базу даних та свій код.
  • Кому треба: Сисадмінам, для "переїзду" своїх серверів у хмару.
  • Приклади: AWS EC2, Azure Virtual Machines.

PaaS — Платформа як послуга

🚀 Що це: Провайдер дає готове середовище для виконання коду (Node.js, Python), налаштовує веб-сервер та ОС.

  • Ви приносите лише свій застосунок і дані.
  • Кому треба: Розробникам, щоб не думати про інфраструктуру.
  • Приклади: AWS Elastic Beanstalk, Heroku.

SaaS — ПЗ як послуга

💻 Що це: Повністю готовий продукт, відкривається в браузері.

  • Провайдер відповідає за все.
  • Кому треба: Кінцевим користувачам.
  • Приклади: Zoom, Microsoft 365, Google Drive, Jira.

4. Типи хмарних середовищ

Публічна та Приватна хмари

Публічна (Public) Приватна (Private)
  • Власник: AWS, Google, Azure.
  • Ресурси спільні для всіх.
  • Найдешевша на старті.
  • Ідеально для стартапів.
  • Власник: Ваша організація.
  • Ізольоване "залізо" (напр., через VMware).
  • Дорога та безпечна.
  • Ідеально для банків та уряду.

Гібридна та Мультихмарна

  • Гібридна: Об'єднує публічну та приватну.
    (Секретні дані у себе, а при пікових навантаженнях сайт підключає AWS — "cloud bursting").
  • Мультихмарна: Використання одразу кількох публічних хмар (напр., AWS + GCP).
    (Щоб не залежати від одного провайдера та брати кращі інструменти у кожного).

Висновки

  • Хмара звільнила ІТ від необхідності купувати дорогі сервери.
  • Лідер ринку — AWS (з 2006 року).
  • Обирайте IaaS для контролю, PaaS для швидкого програмування, SaaS для готового рішення.
  • Моделі розгортання: публічна, приватна, гібридна, мультихмарна.
