1. Віртуальні мережі: VPC та VNet

Ваша "територія" у хмарі

Що таке VPC?

VPC (Virtual Private Cloud) — це ваш ізольований "орендований офіс" у великому датацентрі провайдера. Ви повністю керуєте простором IP-адрес, підмережами і тим, хто має сюди доступ.

Архітектура підмереж

  • Публічна (Public): має доступ до Інтернету через Internet Gateway (IGW). Тут живуть веб-сервери і балансувальники.
  • Приватна (Private): захована від прямого доступу з Інтернету. Вихід в Інтернет (напр. за оновленнями) — лише через NAT Gateway. Тут живуть бази даних і бекенд.

2. Балансування навантаження (Load Balancers)

Розподіл трафіку між серверами

Навіщо це потрібно?

  • Розподіляє 10 000 користувачів на 5 серверів по 2 000 на кожен.
  • Автоматично виключає сервер, якщо той завис (Health Checks).
  • Знімає навантаження з розшифровування HTTPS-трафіку (SSL-термінація).

ALB vs NLB

ALB (Application Load Balancer) NLB (Network Load Balancer)
  • Рівень L7 (працює з HTTP/HTTPS).
  • Розуміє URL. Може відправити запит до /api/ на один сервер, а /images/ — на інший.
  • Ідеальний для веб-сайтів.
  • Рівень L4 (працює з TCP/UDP).
  • Не дивиться всередину запиту. Ультра-швидкий (мільйони запитів в секунду).
  • Ідеальний для ігор, IoT.

3. Міжмережеві екрани (Фаєрволи)

Security Groups (SG)

  • Працюють на рівні одного сервера (віртуальної машини).
  • Stateful: якщо ви дозволили хтось зайти на сервер, відповідь надійшлеться назад автоматично.
  • Логіка: Всі правила є ТІЛЬКИ ДОЗВОЛЯЮЧИМИ. Все, що не дозволено — заборонено.

Network ACL (NACL)

  • Працюють на рівні цілої підмережі.
  • Stateless: треба явно написати правило і для вхідного, і для вихідного трафіку.
  • Логіка: Можуть містити правила "ДОЗВОЛИТИ" і "ЗАБОРОНИТИ" (напр., швидко заблокувати IP хакера для всієї підмережі).

4. CDN та Глобальна маршрутизація

CDN (Content Delivery Network)

Географічно розподілена мережа серверів, яка копіює (кешує) ваш сайт ближче до клієнта.

Користувач з Токіо не чекає поки картинка завантажиться з Києва. Він миттєво отримує її з локального серверу CDN у Токіо.

Приклади: Amazon CloudFront, Cloudflare.

Розумний DNS (Amazon Route 53)

  • Geolocation: відправляє клієнта на сервер у його країні.
  • Latency: вимірює затримку і відправляє клієнта до найшвидшого для нього сервера.
  • Failover: якщо європейський сервер впаде, DNS автоматично перенаправить усіх на запасний сервер в США.

Висновки

  • Бази даних та інші критичні сервіси мають бути приховані у приватній підмережі (Private Subnet).
  • Security Groups (SG) та NACL разом формують потужний дворівневий мережевий захист.
  • CDN — найпростіший спосіб в рази прискорити завантаження сайту для користувачів з усього світу.
