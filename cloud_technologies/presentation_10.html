1. Концепція PaaS

PaaS (Platform as a Service) — платформа як послуга, що надає готове середовище для розробки, розгортання та масштабування застосунків.

Порівняння IaaS vs PaaS

Шар IaaS PaaS
Дані / Застосунок Клієнт Клієнт
Рантайм / Middleware / ОС Клієнт Провайдер
Віртуалізація / Мережа / Обладнання Провайдер Провайдер

Переваги та обмеження

Переваги

  • Швидкість (git push → deploy)
  • Менше DevOps-навантаження
  • Автоматичні патчі безпеки
  • Вбудована масштабованість

Обмеження

  • Менше контролю над ОС/мережею
  • Vendor lock-in
  • Обмеження рантайму (версії мов)
  • Вища ціна при великих масштабах

2. AWS Elastic Beanstalk (EB)

Оркестратор AWS-ресурсів (EC2, ALB, ASG) для веб-застосунків.

  • Підтримка: Node.js, Python, Java, Docker, Go, .NET
  • Автоматичне управління інфраструктурою
  • EB CLI для зручного розгортання

Команди EB CLI


eb init    # Ініціалізація проєкту
eb create  # Створення середовища
eb deploy  # Деплоймент нової версії
eb logs    # Перегляд логів
eb open    # Відкрити у браузері

Blue/Green Deployment

  1. Розгортання нової версії (Green) паралельно до поточної (Blue)
  2. Перевірка працездатності Green-середовища
  3. Переключення DNS зі старого середовища на нове
  4. Результат: нульовий downtime

3. Azure App Service

Високорівнева PaaS-платформа Microsoft.

  • App Service Plan визначає ресурси (Free, Basic, Standard, Premium)
  • Deployment Slots — мабуть, найкраща фіча для Staging/Prod
  • Вбудована Auth/Auth (Azure AD, Google, Facebook)

Deployment Slots & Swap

Можливість миттєвого переключення між Staging та Production:

Production (Live) ↔ Staging (Test)
  • Swap відбувається миттєво без зупинки сервісу
  • Легкий rollback у разі помилок

4. Google App Engine (GAE)

Standard Environment

  • Масштабування до нуля
  • Старт за секунди
  • Обмежений список рантаймів

Flexible Environment

  • Docker-базований
  • Будь-яка мова/бібліотека
  • Мінімум 1 інстанс

Керування через gcloud


gcloud app deploy  # Деплоймент (використовує app.yaml)
gcloud app logs tail # Стрімінг логів
gcloud app versions list # Список версій
gcloud app services set-traffic default --splits v1=0.8,v2=0.2 # A/B testing

5. Критерії вибору

Критерій Elastic Beanstalk Azure App Service App Engine
Складність Середня Низька Середня
Scale to Zero ❌ ❌ ✅ (Standard)
Docker ✅ ✅ ✅ (Flexible)

Trend: Container-based PaaS

  • AWS App Runner — максимально простий деплой контейнера
  • Azure Container Apps — PaaS на базі K8s
  • Google Cloud Run — запуск контейнерів з оплатою за запит

6. Нове покоління PaaS

DX First (Developer Experience)

Пріоритет — максимальне спрощення шляху від коду до продакшну.

  • Zero-config: авто-визначення фреймворку
  • Git-native: push → build → live
  • Preview URLs: кожний PR має свою адресу

Лідери ринку

Vercel: Ідеально для Next.js та React (Edge Network)
Netlify: Jamstack, статичні сайти, форми
Render: Always-on backend, БД, Redis
Railway: Універсальний деплой будь-якого стека

Serverless Бази Даних

  • Neon: Serverless PostgreSQL (масштабування до нуля, БД-гілки)
  • Supabase: PostgreSQL + Auth + Realtime + Storage

Висновки

  • PaaS дозволяє зосередитись на коді, а не на адмініструванні.
  • Elastic Beanstalk, App Service та App Engine — потужні інструменти від "великої трійки".
  • Нове покоління (Vercel, Render) радикально покращує DX та швидкість деплою.
  • Вибір PaaS залежить від стека, бюджету та необхідного рівня контролю.
