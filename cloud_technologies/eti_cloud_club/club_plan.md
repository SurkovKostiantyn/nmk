# Організаційні заходи в роботі гуртка «ETI Cloud Club»

Науково-технічний гурток **«ETI Cloud Club»** на 2024–2025, 2025–2026 навчальні роки є творчим колективом студентів, об'єднаних з метою якісної підготовки фахівців у сфері хмарних технологій, формування у них систематизованих навичок науково-дослідної роботи та умінь застосовувати сучасні хмарні платформи й інструменти в практичній діяльності.

## Організаційні заходи

Для виконання поставлених завдань протягом року гурток проводить такі організаційні заходи:

- організація бесід в академічних групах для залучення студентів до роботи в гуртку;
- проведення пленарних засідань з обговоренням результатів наукових робіт і практичних досліджень у сфері хмарних технологій;
- організація секцій для поглибленого вивчення окремих тематик (безпека, DevOps, архітектура хмарних систем тощо);
- організація наукових конференцій, хакатонів та демо-днів.

## Основні форми роботи

- засідання гуртка з розглядом нових хмарних інструментів та практик;
- зустрічі з представниками IT-компаній та хмарними архітекторами-практиками;
- участь у наукових семінарах, круглих столах і конференціях з хмарних технологій;
- заслуховування звітів студентів за підсумками виконаних практичних завдань;
- виконання науково-дослідних завдань під час навчально-практичного стажування;
- вивчення та узагальнення досвіду провідних хмарних провайдерів (AWS, Azure, GCP);
- підготовка статей, презентацій та технічних звітів;
- розробка рекомендацій і пропозицій щодо впровадження хмарних рішень у навчальний процес.

---

## План роботи на навчальний рік

### 🗓 Вересень — Знайомство та орієнтація

- Знайомство учасників, визначення рівня знань та інтересів.
- Огляд ринку хмарних технологій: AWS, Azure, GCP — порівняння платформ.
- Введення в концепції IaaS, PaaS, SaaS.

### 🗓 Жовтень — Основи хмарних обчислень

- Реєстрація у хмарних платформах, знайомство з консолями управління.
- Практична робота: розгортання першої віртуальної машини в AWS / Azure.
- Огляд базових сервісів: обчислення, зберігання, мережа.

### 🗓 Листопад — Зберігання та бази даних у хмарі

- Хмарні сховища: S3, Azure Blob Storage, Google Cloud Storage.
- Хмарні бази даних: RDS, Azure SQL, Cloud SQL, DynamoDB.
- Практичне завдання: завантаження і доступ до даних через хмарне сховище.

### 🗓 Грудень — Безпека у хмарних середовищах

- Shared Responsibility Model: розподіл відповідальності.
- IAM: ролі, групи, принцип мінімальних привілеїв, MFA.
- Практичне завдання: налаштування IAM-ролей для тестового проєкту.

### 🗓 Січень — Контейнеризація

- Введення в Docker: образи, контейнери, Docker Compose.
- Практичне завдання: контейнеризація власного застосунку.
- Огляд Container Registry: ECR, ACR, GCR.

### 🗓 Лютий — Оркестрація та DevOps

- Основи Kubernetes: Pod, Deployment, Service.
- CI/CD у хмарі: GitHub Actions, AWS CodePipeline, Azure DevOps.
- Практичне завдання: побудова простого CI/CD pipeline.

### 🗓 Березень — Хмарна архітектура та практичний проєкт

- Serverless: AWS Lambda, Azure Functions, Google Cloud Functions.
- Мікросервісна архітектура та API Gateway.
- Старт роботи над груповим проєктом із хмарної інфраструктури.

### 🗓 Квітень — Демонстрація проєктів

- Завершення та презентація групових проєктів.
- Отримання зворотного зв'язку від учасників та викладачів.
- Огляд сертифікаційних програм: AWS Certified Cloud Practitioner, AZ-900, ACE.

### 🗓 Травень — Підсумки та планування

- Обговорення досягнень і прогресу учасників.
- Підготовка звіту про діяльність гуртка.
- Планування тем і форматів роботи на наступний навчальний рік.
