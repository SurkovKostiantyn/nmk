#### 1.5 Робота з базами даних та клієнтами (GUI)

Жоден сучасний веб-застосунок не може існувати без системи зберігання даних. Якщо код — це "мозок" програми, то **база даних (БД)** — це її "пам'ять". Як розробнику, вам важливо вміти не лише писати запити, а й розгортати та адмініструвати середовища для різних типів БД.

##### 1.5.1 Локальні веб-сервери та збірки: використання XAMPP

Для новачків найпростішим способом отримати робоче середовище є використання збірки **XAMPP**. Вона автоматично встановлює та налаштовує зв'язку Apache + MariaDB (аналог MySQL) + PHP.

Переваги використання XAMPP:

- **Швидкий старт** — база даних готова до роботи натисканням однієї кнопки "Start" у панелі керування;
- **phpMyAdmin** — вбудований веб-інтерфейс для керування базами даних, доступний за адресою `http://localhost/phpmyadmin/`;
- **Консолідація** — всі логи та конфігурації зібрані в одному місці.

##### 1.5.2 Реляційні бази даних: встановлення та налаштування MySQL

**MySQL** — це найпопулярніша реляційна система управління базами даних (СУБД) для веб-розробки. Вона використовує мову **SQL (Structured Query Language)** для маніпуляції даними.

Основні моменти при налаштуванні:

- **Порт за замовчуванням** — 3306. Переконайтеся, що він не зайнятий іншими програмами (наприклад, Skype);
- **Користувач root** — адміністратор за замовчуванням. У локальних середовищах він часто не має пароля, але в "бойових" умовах це критична діра в безпеці;
- **Кодування** — завжди використовуйте `utf8mb4` для повної підтримки Unicode (включаючи емодзі).

##### 1.5.3 Робота з SQLite як вбудованою базою даних

**SQLite** — це унікальна СУБД, яка не потребує окремого сервера. Вся база даних — це один файл на диску.

Чому це зручно:

- **Нульова конфігурація** — не потрібно встановлювати сервіси, додавати користувачів чи відкривати порти;
- **Портативність** — файл бази даних можна просто перенести на інший комп'ютер разом із кодом проєкту;
- **Ідеально для dev-середовища** — PHP та Python мають вбудовану підтримку SQLite "з коробки", що робить її ідеальною для швидкого прототипування.

##### 1.5.4 Клієнти для візуальної роботи з БД (GUI)

Хоча Senior-розробники часто пишуть запити вручну, для швидкого перегляду даних та редагування структури таблиць використовуються графічні інтерфейси.

Рекомендовані інструменти:

- **DBeaver** — безкоштовний універсальний клієнт, що підтримує майже всі відомі БД (MySQL, PostgreSQL, SQLite, MongoDB);
- **DataGrip** — професійний інструмент від JetBrains (платний), який має найкращий у класі автодоповнювач коду та інструменти рефакторингу SQL;
- **MySQL Workbench** — офіційний інструмент від розробників MySQL, найкращий для моделювання складних ER-діаграм.

##### 1.5.5 Встановлення та налаштування PostgreSQL

**PostgreSQL** вважається найбільш просунутою реляційною БД з відкритим кодом. Вона пропонує більше можливостей, ніж MySQL, і часто обирається для складних корпоративних систем.

Особливості PostgreSQL:

- **Тип даних JSONB** — дозволяє ефективно працювати з неструктурованими даними всередині реляційної таблиці;
- **Розширюваність** — підтримка географічних даних (PostGIS) та повнотекстового пошуку;
- **Надійність** — суворе дотримання стандартів ACID (атомарність, узгодженість, ізольованість, довговічність).

##### 1.5.6 Поняття NoSQL баз даних: MongoDB

На відміну від реляційних БД, **NoSQL** системи (зокрема MongoDB) зберігають дані не в таблицях, а у вигляді документів (схожих на JSON-об'єкти).

Робота з MongoDB:

- **Гнучка схема** — ви можете додавати нові поля до документів без необхідності змінювати структуру всієї таблиці (Schema-less);
- **MongoDB Compass** — офіційний GUI-клієнт, який візуалізує дані та допомагає будувати складні агрегаційні запити;
- **Масштабованість** — архітектура MongoDB спроектована для легкого горизонтального масштабування на багато серверів.
