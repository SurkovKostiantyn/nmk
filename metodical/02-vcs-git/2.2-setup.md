#### 2.2 Налаштування локального середовища Git

Перед початком роботи необхідно підготувати ваш комп'ютер: встановити саму програму Git та ідентифікувати себе як автора майбутніх змін.

##### 2.2.1 Встановлення клієнта Git на робочу машину

Для більшості розробників на Windows стандартним вибором є **Git for Windows**. Разом із ним ви отримуєте не лише консольну утиліту, а й **Git Bash** — емулятор терміналу Linux, який ми рекомендували у Розділі 1.

Кроки встановлення:

- **Завантаження** — перейдіть на [git-scm.com](https://git-scm.com/) та завантажте останню версію;
- **Налаштування редактора** — під час інсталяції оберіть Visual Studio Code як редактор за замовчуванням для Git;
- **Глобальні налаштування** — залиште значення за замовчуванням для більшості параметрів (особливо щодо завершення рядків `LF/CRLF`).

##### 2.2.2 Конфігурація користувача

Git додає інформацію про автора до кожного вашого коміту. Це дозволяє команді знати, хто саме вніс конкретні зміни.

Виконайте наступні команди у терміналі (замінивши дані на власні):

```bash
git config --global user.name "Ivan Ivanov"
git config --global user.email "ivan.ivanov@example.com"
```

Перевірити поточні налаштування можна командою `git config --list`.

##### 2.2.3 Безпека у спільних аудиторіях

При роботі на комп'ютерах у навчальних лабораторіях ви повинні бути надзвичайно обережними зі своїми даними. **Пам'ятайте: ваш акаунт GitHub — це ваша цифрова репутація.**

Правила безпеки:

- **Вихід з акаунту** — завжди розлогінюйтесь із GitHub у браузері після завершення пари;
- **Менеджер паролів** — ніколи не погоджуйтесь на пропозицію браузера "Зберегти пароль";
- **Тимчасові файли** — якщо ви створювали SSH-ключі без пароля, видаліть їх із папки `~/.ssh/` перед тим, як залишити робоче місце.

##### 2.2.4 Налаштування SSH-ключів для комунікації з GitHub

**SSH (Secure Shell)** — це основний спосіб безпечної взаємодії з віддаленими репозиторіями. Він дозволяє вам працювати з GitHub без постійного введення логіна та пароля.

Процес налаштування:

1. **Генерація ключа** — виконайте команду `ssh-keygen -t ed25519 -C "ваш_email"`. На всі запитання просто натискайте `Enter`;
2. **Копіювання публічного ключа** — відкрийте файл `~/.ssh/id_ed25519.pub` та скопіюйте його вміст;
3. **Додавання в GitHub** — перейдіть у _Settings -> SSH and GPG keys -> New SSH key_ та вставте скопійований текст.

Перевірка зв'язку:

```bash
ssh -T git@github.com
```

##### 2.2.5 Робота з HTTPS-доступом (Personal Access Tokens)

Якщо ви не можете використовувати SSH, ви можете працювати через HTTPS. Однак із 2021 року GitHub більше не приймає звичайні паролі від акаунту для операцій Git.

Замість пароля необхідно використовувати **PAT (Personal Access Token)**:

- **Створення** — генерується в _Settings -> Developer settings -> Personal access tokens_;
- **Область дії** — при створенні вказуйте лише необхідні права (scope), наприклад `repo`.

##### 2.2.6 Встановлення та використання GitHub CLI (`gh`)

**GitHub CLI (`gh`)** — це офіційна консольна утиліта, яка переносить можливості веб-інтерфейсу GitHub прямо у ваш термінал.

Переваги `gh`:

- **Авторизація** — команда `gh auth login` дозволяє швидко налаштувати доступ через браузер;
- **Швидкість** — створення репозиторіїв, форків та Pull Request-ів відбувається однією командою без перемикання вікон;
- **Зручність** — ви можете переглядати статус CI/CD перевірок безпосередньо в консолі.
