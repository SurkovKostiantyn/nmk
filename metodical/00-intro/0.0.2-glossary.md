# СЛОВНИК ТЕРМІНІВ (ГЛОСАРІЙ)

У цьому посібнику використовуються такі ключові терміни та поняття:

- **Бекенд (Backend)** — серверна частина веб-додатку, яка зазвичай відповідає за роботу з базою даних, системну логіку та обробку запитів від клієнта.
- **Гілка (Branch)** — у системах контролю версій (наприклад, Git) це незалежна лінія розробки. Використовується для роботи над новою функціональністю або виправленням помилок ізольовано від основного коду (зазвичай гілки `main` або `master`).
- **Дамп (Dump)** — файл, що містить повну копію структури та даних бази даних для резервного копіювання або перенесення на інший сервер.
- **Деплой / Розгортання (Deployment)** — процес перенесення програмного забезпечення або веб-додатку з локального середовища розробника на бойовий (або тестовий) сервер, де воно стає доступним кінцевим користувачам.
- **Динамічна типізація (Dynamic Typing)** — властивість мови програмування (наприклад, JavaScript, Python, PHP), коли типи змінних визначаються автоматично під час виконання програми.
- **Ендпоінт (Endpoint)** — конкретна адреса (URL) в API, за якою сервер приймає запити та повертає відповідні дані.
- **Залежності (Dependencies)** — зовнішні бібліотеки або пакети (наприклад, з npm, Composer чи pip), від яких залежить правильна робота додатка.
- **Змінна середовища (Environment Variable)** — динамічне значення, що впливає на поведінку обчислювального процесу, наприклад підключення до бази даних, API-ключі тощо. Часто зберігається у файлі `.env`.
- **Інтерфейс програмування застосунків (API)** — набір правил і протоколів, що дозволяє різним програмам або сервісам спілкуватися між собою.
- **Клієнт (Client)** — програма або пристрій (наприклад, веб-браузер або мобільний додаток), що відправляє запити до сервера та відображає отримані дані користувачу.
- **Клонування (Cloning)** — у контексті Git: створення локальної копії віддаленого репозиторію на вашому комп'ютері.
- **Коміт (Commit)** — у системах контролю версій: зафіксований стан або знімок змін у файлах репозиторію в певний момент часу з відповідним описом того, що саме було зроблено.
- **Консоль / Термінал (Console / Terminal)** — текстовий інтерфейс для взаємодії з операційною системою за допомогою командного рядка.
- **Кеш (Cache)** — тимчасове сховище даних, яке використовується для прискорення доступу до інформації, яку часто запитують (наприклад, кешування веб-сторінок).
- **Лінтер (Linter)** — інструмент статичного аналізу коду, який допомагає знаходити синтаксичні або стилістичні помилки в програмному коді (наприклад, ESLint).
- **Майстер / Мейн (Master / Main)** — назва основної (головної) гілки у системі контролю версій Git, у якій зберігається робочий і перевірений код.
- **Маршрутизація (Routing)** — процес визначення шляху (URL), за яким веб-додаток обробляє специфічні запити користувача та повертає відповідну сторінку або дані.
- **Мідлвар (Middleware)** — проміжне програмне забезпечення, яке обробляєHTTP-запити до того, як вони досягнуть основного контролера (наприклад, для перевірки авторизації).
- **Міграція (Migration)** — спосіб керування та документування змін у структурі бази даних за допомогою коду, що дозволяє легко оновлювати або відкочувати схему БД.
- **Нода / Вузол (Node)** — будь-який пристрій (комп'ютер, сервер), підключений до комп'ютерної мережі; або екземпляр середовища виконання (наприклад, процес Node.js).
- **Пакетний менеджер (Package Manager)** — інструмент, що автоматизує процеси встановлення, оновлення та налаштування програмного забезпечення або сторонніх бібліотек (наприклад, npm для JavaScript, Composer для PHP, pip для Python).
- **Парсинг (Parsing)** — процес аналізу та перетворення структурованого тексту або даних (наприклад, JSON, XML) у формат, зрозумілий для програмного коду (в об'єкти або масиви).
- **Плагін / Розширення (Plugin / Extension)** — додатковий програмний компонент, який додає специфічні можливості до більшого додатка (наприклад, плагіни для VS Code).
- **Реквест / Запит (Request)** — повідомлення, яке клієнт відправляє на сервер, щоб отримати дані або виконати певну дію (наприклад, GET або POST запити).
- **Репозиторій (Repository)** — місце, де зберігаються файли проєкту (код, графіка тощо) та вся історія їх змін (у системі контролю версій).
- **Сервер (Server)** — комп'ютер (або програма), який надає послуги іншим програмам або пристроям (клієнтам). Наприклад, веб-сервер (Apache, Nginx) обробляє HTTP-запити від браузерів.
- **Серіалізація (Serialization)** — процес перетворення об'єктів або структур даних з пам'яті програми у формат, який можна зберегти у файл або передати по мережі (наприклад, перетворення об'єкта під час передачі у JSON-рядок).
- **Синтаксис (Syntax)** — набір правил певної мови програмування, які визначають, як правильно писати або структурувати команди у цій мові.
- **Статична типізація (Static Typing)** — властивість мови програмування (наприклад, TypeScript, Java, C++), коли типи змінних визначаються та жорстко перевіряються на етапі компіляції або написання коду.
- **Токен (Token)** — унікальний рядок символів, що використовується для ідентифікації або авторизації користувача в системах (наприклад, JWT – JSON Web Token).
- **Форк (Fork)** — створення власної незалежної копії чужого репозиторію на платформі на кшталт GitHub або GitLab для подальшого внесення власних змін та експериментів.
- **Фреймворк (Framework)** — набір готових програмних компонентів, паттернів архітектури та правил, який задає структуру для швидшого створення додатків (наприклад, React/Vue для фронтенду, Laravel/Django для бекенду).
- **Фронтенд (Frontend)** — клієнтська (видима) частина веб-додатку, з якою безпосередньо взаємодіє користувач. Будується з використанням HTML, CSS та JavaScript.
- **Хостинг (Hosting)** — послуга надання ресурсів для розміщення інформації на сервері, що постійно підключений до інтернету.
