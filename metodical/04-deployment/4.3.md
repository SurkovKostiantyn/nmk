##### 4.3 Загальні поняття хостингу та Cloud-сервісів

Щоб ваш веб-застосунок став доступним для користувачів з усього світу, він має бути розміщений на сервері, який працює цілодобово і має публічну IP-адресу. Ця послуга називається **хостингом**.

У сучасному вебі традиційні сервери все частіше замінюються **хмарними сервісами (Cloud Services)**, які пропонують більшу гнучкість та надійність.

Основні сервісні моделі хмарних обчислень:

- **IaaS (Infrastructure as a Service)** — ви орендуєте "залізо" (віртуальні сервери, сховища). Ви самі встановлюєте ОС та налаштовуєте все середовище (наприклад, AWS EC2, DigitalOcean Droplets);
- **PaaS (Platform as a Service)** — провайдер надає готову платформу з налаштованою ОС, базою даних та середовищем виконання (Node.js, PHP). Ви просто завантажуєте свій код (наприклад, Heroku, Vercel, Google App Engine);
- **SaaS (Software as a Service)** — готова до використання програма, доступна через браузер. Вам не потрібно нічого розгортати (наприклад, Google Docs, Slack, GitHub).

Популярні хмарні провайдери:

- **Глобальні гіганти** — Amazon Web Services (AWS), Google Cloud Platform (GCP), Microsoft Azure. Вони пропонують сотні сервісів для будь-яких задач;
- **Для розробників** — DigitalOcean, Linode, Hetzner. Пропонують простіші та дешевші VPS (Virtual Private Servers);
- **Спеціалізовані (Serverless/Static)** — Vercel, Netlify, GitHub Pages. Оптимізовані для швидкого розгортання сучасних фронтенд-фреймворків та статичних сайтів.

> 🖼️ [МІСЦЕ ДЛЯ СХЕМИ: Порівняння рівнів контролю: On-Premises (ви керуєте всім) -> IaaS -> PaaS -> SaaS (провайдер керує всім)]

Для навчальних проєктів ми найчастіше будемо використовувати модель **PaaS**, оскільки вона дозволяє сфокусуватися на коді, а не на адмініструванні серверів Linux.
