##### 4.2 Базове використання GitHub Actions

**GitHub Actions** — це інструмент автоматизації, який дозволяє виконувати певні дії (скрипти) безпосередньо у хмарі GitHub у відповідь на події у вашому репозиторії.

Основні поняття:

- **Workflow (Робочий процес)** — автоматизована процедура, описана у YAML-файлі, яка складається з однієї або декількох робіт;
- **Event (Подія)** — тригер, що запускає процес (наприклад, `push` коду або створення `pull_request`);
- **Job (Робота)** — набір кроків, які виконуються на одному і тому самому віртуальному сервері (Runner);
- **Step (Крок)** — окрема задача, наприклад, встановлення залежностей або запуск скрипта.

Конфігураційні файли завжди зберігаються в папці `.github/workflows/`.

Приклад найпростішого Workflow для перевірки коду (лінтування):

```yaml
name: Lint and Test
on: [push, pull_request]

jobs:
  check-code:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: "20"

      - name: Install dependencies
        run: npm install

      - name: Run Linter
        run: npm run lint
```

Як це працює на практиці:

1. Ви робите коміт і `git push`;
2. GitHub бачить нову подію і запускає віртуальну машину з Ubuntu;
3. Виконуються кроки: завантажується ваш код, встановлюється Node.js та бібліотеки;
4. Запускається лінтер. Якщо він знайде помилки — робота завершиться зі статусом **Failed**.

Використання GitHub Actions дозволяє вам бути впевненими, що в основну гілку ніколи не потрапить код, який не проходить тести або має погане форматування.
